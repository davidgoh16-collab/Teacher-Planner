## 2025-04-11 - Hardcoded Firebase API Key
**Vulnerability:** A critical Firebase API Key (`AIza...`) was hardcoded directly into `firebase.ts`, exposing the project's database to potential unauthorized access or manipulation if the source code were leaked or decompiled.
**Learning:** For Vite React applications deployed to Google Cloud Run via direct git deployment (using a Docker container with an Nginx reverse proxy), environment variables must be passed down multiple layers: via Dockerfile `ARG`/`ENV`, injected via `start-nginx.sh` into an `env.js` file, and exposed globally through a typed `window.ENV` interface. Relying on hardcoded strings bypasses this secure configuration pipeline.
**Prevention:** Always use `window.ENV?.VITE_FIREBASE_API_KEY || import.meta.env.VITE_FIREBASE_API_KEY` for configuration secrets. Ensure new secrets are added to the Dockerfile, `start-nginx.sh`, and `types.ts` to maintain secure, environment-specific injection at container startup.
